Titing

把 AI 工程执行，变成一套清晰、可治理的系统。

蹄听 是一个 local-first 的 AI 工程执行控制器，负责任务生命周期、调度、执行、评测、Goal Loop、观测与治理。

Overview

它解决什么问题

当 AI 开始真正参与工程交付，问题不再只是“能不能生成代码”，而是任务如何接入、怎么调度、何时停止、谁来接管，以及整个过程能不能被追踪和治理。

Titing 把这些问题收敛到同一套运行时里，让任务从接入到交付都沿着明确的状态机推进，而不是散落在脚本、临时约定和人工操作之间。

Capabilities

核心能力

  • 统一任务生命周期管理：创建、调度、执行、评测、修复、人工接管。
  • local-first 运行方式：围绕本地 worktree、SQLite 与 CLI 执行器组织执行。
  • 插件化扩展：接入、环境、执行、质量、治理都通过稳定契约解耦。
  • 质量与风险控制：支持 lint、类型检查、测试、构建与 diff 风险评估。

Flow

执行流程

  1. 接入任务：手动创建、轮询或 Webhook 统一进入同一任务模型。
  2. 调度执行：根据插件健康度、执行器能力和环境状态分配任务。
  3. 运行评测：执行工作流，随后进入质量检查与结果评估。
  4. 修复或停机：失败进入 repair loop；高风险或重复失败时转人工接管。
